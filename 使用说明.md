# 使用说明

## 文件内容

- `README.md`：完整 GitHub 个人主页
- `.github/workflows/snake.yml`：每天自动生成贡献贪吃蛇动画

## 推荐部署方式

将这两个文件复制到你的特殊主页仓库：

```text
refuse-wrong/refuse-wrong
```

目录结构：

```text
refuse-wrong/
├── README.md
└── .github/
    └── workflows/
        └── snake.yml
```

提交并推送：

```bash
git clone https://github.com/refuse-wrong/refuse-wrong.git
cd refuse-wrong

# 使用本方案中的文件覆盖仓库文件
git add README.md .github/workflows/snake.yml
git commit -m "feat: redesign GitHub profile"
git push origin main
```

首次推送后：

1. 打开仓库的 Actions 页面。
2. 进入 `Generate Contribution Snake`。
3. 点击 `Run workflow`。
4. 等待 `output` 分支生成。
5. 刷新个人主页。

## 需要检查的内容

- 邮箱是否希望公开；不希望公开可删除两个 Email 按钮。
- 个人网站目前使用 `https://refuse-wrong.github.io`。
- GitHub 统计卡片由第三方服务动态生成，偶尔可能加载较慢。
- 建议在 GitHub 个人资料中固定 6 个代表性仓库。


## 最近 GitHub 动态自动更新

新增文件：

```text
.github/workflows/recent-activity.yml
.github/recent-activity.config.yml
```

该工作流每 6 小时更新一次 README 中的：

```text
<!--START_SECTION:activity-->
<!--END_SECTION:activity-->
```

它会展示最近的提交、Pull Request、Issue、代码审查等 GitHub 动态。

首次使用时，在仓库 Actions 页面手动运行：

```text
Update GitHub Activity
```

如果工作流无法写入 README，请前往：

```text
Settings → Actions → General → Workflow permissions
```

选择：

```text
Read and write permissions
```
